Бэк сделать на vertx
Взаимодействие с фронтом через event bus 
Бэк должен уметь:
- Обрабатывать команды из консоли ssh
- Сохранять и загружать состояние сессий
- Обрабатывать ошибки и исключения
- Логировать все действия и ошибки
- Сессия должна поддерживать сохранение состояния и восстановление после перезапуска
Когда создаётся запрос с фронта, должен передаваться идентификатор сессии, чтобы можно было связать запрос с конкретной сессией и восстановить состояние при необходимости.
Множество вкладок с одним и тем же ssh подключением, должно синхронизироваться между ними, чтобы изменения в одной сессии отображались в других. (посредством event bus)
При открытии новой вкладки, если сессия уже существует, должно восстановиться состояние сессии из базы данных (redis)
Состояние это - подключенные ssh сессии.

Визуально терминал на фронте должен представлять из себя табы с возможностью переключения между ними и синхронизацией состояния между вкладками.
